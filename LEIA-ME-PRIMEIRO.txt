JUMP METRICS — ARQUIVOS PARA PUBLICAR NO GITHUB

Esta pasta já está pronta para ser enviada ao GitHub Pages.
Os arquivos usados pelo site estão todos aqui, na pasta principal:

- index.html
- 404.html
- app.js
- app.css
- config.js
- CNAME
- .nojekyll

ANTES DE ENVIAR AO GITHUB
1. Abra o arquivo config.js com o Bloco de Notas.
2. Cole a Project URL e a Publishable Key do Supabase:

window.JUMP_METRICS_CONFIG = {
  SUPABASE_URL: 'https://SEU-PROJETO.supabase.co',
  SUPABASE_PUBLISHABLE_KEY: 'sb_publishable_SUA_CHAVE',
};

3. Salve o arquivo.
4. No GitHub, crie um repositório chamado jump-metrics.
5. Envie TODOS os arquivos desta pasta para a raiz do repositório.
6. No repositório, abra Settings > Pages.
7. Em Build and deployment, escolha Deploy from a branch.
8. Escolha a branch main e a pasta /(root), depois clique em Save.
9. Em Custom domain, informe jumpmetrics.com.br.

ATENÇÃO
- Pode usar a Publishable Key no config.js.
- Nunca use Secret Key, service_role ou senha do banco neste arquivo.
- O arquivo de banco SQL é separado e deve ser executado no Supabase, não no GitHub.
